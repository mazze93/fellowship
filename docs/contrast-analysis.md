# Contrast Analysis

## Validation Protocol

Dark environments (Signal · Moonlight, Forge, Hearth, Havens · Moonlight) should be
validated in a low-light environment at approximately 30% display brightness.

## Check For

- Gradient banding — inspect at full resolution in a dark room
- Highlight clipping — verify accent highlights don't blow out
- Menu bar interference — confirm accent colors remain legible against system chrome
- Dock interference — verify Dock icon visibility is not degraded

## Per-Environment Notes

| Environment | Key Risk | Mitigation |
|---|---|---|
| Signal · Day | Drowsiness from soft contrast | Soft mid contrast — not flat; teal accent maintains alerting presence |
| Signal · Moonlight | Same as Signal · Day at lower brightness | Dim teal, low contrast; avoid further reducing display brightness |
| Drift | Eye fatigue from high luminance | High luminance is intentional; limit session length or use with breaks |
| Forge | Tunnel collapse — missed periphery | Narrow highlight is deliberate; ensure menu bar remains accessible |
| Hearth | Visual monotony over long sessions | Gentle contrast variation across elements; monitor for fatigue after 90+ min |
| Havens · Day | Over-softening reduces legibility | Luminance kept high enough to distinguish interactive elements |
| Havens · Moonlight | Overstimulating highlights disrupt wind-down | No bright accents; cool blue-silver palette kept dim |

## Color Profile

All exports use sRGB IEC61966-2.1. Validate with a calibrated display.
